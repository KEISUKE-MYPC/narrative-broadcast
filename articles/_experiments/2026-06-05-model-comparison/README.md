# モデル比較実験：BTC 6hナラティブ分析（2026-06-05）

同一データ・同一プロンプトで、Ollama cloudの4モデルにBTC 6h分析を書かせ、Opus 4.8（ルーチン本番モデル）と比較した記録。

- **固定入力**：`00-prompt-used.txt`（2026-06-05 09:06 JST取得の実データスナップショット＋方法論ルール）
- **基準**：`opus-4.8-baseline.md`（同時点のルーチン自動生成・9:06版）
- **条件**：Ollama API `stream:false`、ローカル1回実行。乱数・時点で揺れるため「この回の傾向」として扱う。

## 評価

全モデルが合格点：3層分析・Reflexivity判定・前回比較・免責文をクリア、価格予想/売買助言への逸脱なし。指示追従性は総じて高い。

**Opusの優位点**：「cheap talkの二重化」＝強気（6桁回復）だけでなく新台頭の『底・買い』フレームも資金（ステーブル収縮・OI減）を伴わない、と両側を斬る独自洞察。他モデルにはこの切れ味は出なかった。

**各モデル所感**
- **deepseek-v4-pro** 🥇：強度判定がOpusと一致（−1、理由づけも同型）。「前回はX/今回はY」の対比が方法論に忠実。文章も最も自然。4つで最もOpusに近い。
- **minimax-m3**：内容は厚く、ステーブル流出を「外生的流動性収縮」と的確に。ただしソロス局面①〜④を当てはめる定型感。強度±0でやや弱気維持寄り。
- **kimi-k2.6**：「相転移前夜の均衡破れ」等の framing が上手い。ただし「逆張き」（逆張りの誤字）が複数回＝日本語細部に難。
- **gemma4:31b**：圧倒的に速い（23s）＆読みやすい。が強度を+1と判定し、他3＋Opusの「弱気の確信が後退（−1〜±0）」と逆方向の読み。最小モデルゆえ健闘だが解釈の方向を外した。

## 結論
- 品質でOpusに最も近いのは **deepseek-v4-pro**、次いで minimax。日本語精度は deepseek > minimax > gemma > kimi。
- 速度・コスト重視なら **gemma4:31b**（23秒）だが、ナラティブの“方向”を読み違えるリスクがこの回では出た。
- Opus級の独自洞察は他モデルでは出にくいが、deepseek/minimaxは実用十分。

- 1回実験のため確定的でない。傾向を固めるなら複数回実行して勝率を取る。
- ルーチン本番はOpusのまま（この実験では変更していない）。
