# タイトル長の是正（ハイブリッド） — 設計書

作成日: 2026-06-14

## 背景・課題

記事の見出し（記事H1＝INDEXの`narrative`＝`<title>`/og/カード/JSON-LD headline の共通の出所）が
極端に長い。実測（39記事）：**中央値228字・最長1013字・40字超97%・60字超69%**。

原因：`pipeline/build-prompt.ts` のプロンプトが「タイトルは支配ナラティブを1行で」と
指示するのみで**文字数上限が無い**ため、nemotronが見出しに分析の主張文を書いてしまう。
本文には別途「## 現在の支配的ナラティブ」で詳しい主張があるため、見出しが短くても情報は失われない。

検索結果（日本語で約30字表示）に収まらず、CTR・伝達の面で不利。

## ゴール

検索結果に収まる適切な長さの見出し／`<title>` を実現する。
方針は**ハイブリッド**（パイプラインで未来を根治・サイト側で既存を遡及救済）。

決定事項（2026-06-14）:
- 見出しの文字数上限は **全角40字**。
- **オンページのH1・カード見出しは変更しない**（既存記事の見出しは編集コンテンツとして温存。
  未来の記事はパイプライン側で短くなる）。サイト側短縮は `<title>` 系メタのみ。

## 対応

### ① パイプライン側（未来の見出しを根治） — `pipeline/build-prompt.ts`

- 見出し指示に上限を明記。現行のガードレール
  「6. タイトルは支配ナラティブを1行で。…」を
  「6. タイトルは支配ナラティブを**全角40字以内**で。詳細な論証は本文に書く。…（既存の体裁語禁止は維持）」へ。
- 出力形式の注記 `# タイトル（支配ナラティブを1行・煽らない）` を
  `# タイトル（支配ナラティブを全角40字以内・煽らない）` へ。
- これで今後の記事はH1・カード・`<title>`・og すべてが自動で適正化される（全てH1由来）。
- LLMの遵守は完璧でないため、下の②が安全網。

### ② サイト側（既存39記事＋保険） — `lib/title.ts`（新規）

`shortTitle(narrative: string, max = 40): string`
- `narrative` が `max` 字以内ならそのまま返す。
- 超える場合、`max` 字以内に収まる**最後の自然な区切り**（`、`『。』`／`『・』`―`『:』` 等の区切り文字）で切る。
  - 区切りが `max` 字以内に1つも無ければ `max` 字でハード切り。
  - いずれの場合も末尾に `…` を付す（元が `max` 字以内のときは付けない）。
- 純粋関数（I/Oなし）。

適用先（見出し由来のメタを短縮版へ差し替え） — `app/articles/[...slug]/page.tsx`：
- `generateMetadata` の `title` / `openGraph.title` / `twitter.title`
- `articleJsonLd` の `headline`（`shortTitle` 適用後を渡す）

> `articleJsonLd` 呼び出し時に `title` 引数へ `shortTitle(row.narrative)` を渡す形にする
> （`lib/seo.ts` の `articleJsonLd` 自体は変更不要）。

### 変更しないもの
- オンページの記事H1（`renderMarkdown` 出力の `<h1>`）、カード見出し（`ArticleList` の `ac-title`）、
  INDEXの`narrative`。これらは既存記事では長いまま表示されるが、未来記事は①で短くなる。

## データフロー

- 未来：nemotron が40字以内のH1生成 → INDEX/オンページ表示/`<title>`/og/JSON-LD すべて短い。
- 既存：オンページH1は長いまま／`<title>`・og・JSON-LD headline は `shortTitle` で短縮されSEO最適。

## エラー処理

- `shortTitle` は空文字や非常に短い入力でも安全（そのまま返す）。例外を投げない。

## テスト

`lib/title.test.ts`（Vitest）:
- `max` 字以内はそのまま（`…` を付けない）。
- 区切り文字での短縮（区切り位置で切れ、`…` が付く）。
- 区切りが無い長文はハード切り＋`…`（長さ = max + 1）。
- 境界値（ちょうど `max` 字）。

パイプラインのプロンプト変更は文言のため専用テストは設けないが、
既存の `pipeline/build-prompt.test.ts` が壊れないことを確認する。

## 検証方法

1. `npm test` 全パス、`npx tsc --noEmit` clean、`npm run build` 成功。
2. ビルド済み記事HTMLの `<title>` が「短縮見出し | Narrative Broadcast」形式で40字台に収まることを確認
   （長文narrativeの記事で `<title>` 長が大幅短縮されること）。
3. `articleJsonLd` の `headline` が短縮版になっていること（HTML内 JSON-LD を確認）。
4. オンページのH1は従来どおり（既存記事で変化なし）。
5. （手動・任意）プロンプト変更後の次回自動公開記事で、生成H1が40字前後に収まることを確認。

## スコープ外

既存記事のH1/本文の書き換え・再生成、カード見出しの短縮表示、INDEXの再生成。
